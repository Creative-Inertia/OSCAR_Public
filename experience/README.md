# Experience Files

This folder is where you store learnings from using OSCAR with Fusion 360. As you discover API quirks, develop modeling recipes, and learn what works, capture them here so OSCAR gets smarter over time.

## Format

Each file should use this structure:

```markdown
---
name: Short descriptive name
type: lesson | recipe | reference | tip
tags: [relevant, tags, here]
---

**What:** One-line summary of the learning.

**Details:** Full explanation, code snippets, or step-by-step recipe.

**Why it matters:** What goes wrong if you don't know this.
```

## Types

- **lesson** — Something that went wrong and what you learned (e.g., "hidden bodies are silently skipped by cut operations")
- **recipe** — A working step-by-step procedure for a specific task (e.g., "how to create a coil spring")
- **reference** — API details or patterns worth remembering (e.g., "how to rename a document via API")
- **tip** — A small but useful technique (e.g., "always verify parametric mode on new documents")

## Sharing

If you've discovered something useful, consider submitting a PR to the OSCAR repo. Community-contributed experience files help everyone build better.
